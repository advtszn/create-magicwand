# Support

## Getting help

Use GitHub issues for bugs, documentation problems, and feature requests.

Before opening an issue:

- make sure you are using the latest published version when possible
- include the command you ran
- include the selected runtime, framework, and toolchain
- include relevant error output or screenshots

## Good issue reports

A good report usually includes:

- expected behavior
- actual behavior
- reproduction steps
- environment details such as Bun and Node versions

## Security issues

For vulnerabilities, use the process in `SECURITY.md` instead of a public issue.

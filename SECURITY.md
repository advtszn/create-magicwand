# Security Policy

## Reporting a vulnerability

Do not open public GitHub issues for security problems.

Use GitHub Security Advisories for private reporting if available for this
repository. If that is not available, contact the maintainer directly and share:

- a description of the issue
- affected package or template
- reproduction steps
- impact and any suggested mitigation

## What to expect

We will try to acknowledge reports promptly, validate the issue, and coordinate
a fix before public disclosure.

## Scope

Security reports are most useful when they affect:

- the published CLI in `packages/cli`
- shipped templates under `packages/cli/templates/**`
- release or distribution behavior
